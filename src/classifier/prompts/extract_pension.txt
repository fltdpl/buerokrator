Antworte ausschließlich mit JSON.
Verwende keine Arrays.
Verwende keine Listen.
Liefere ausschließlich die Felder aus dem Schema.
Keine zusätzlichen Felder.

Schema:

{{
  "issuer": "",
  "product_name": "",
  "policy_number": "",
  "document_date": "",
  "document_subtype": "",
  "amount": null,
  "interest": null,
  "capital_gains_tax": null,
  "soli": null,
  "church_tax": null,
  "contributions_total": null,
  "opening_balance": null,
  "closing_balance": null
}}

Extrahiere:

- issuer = Anbieter der Rentenversicherung
- product_name = Produktbezeichnung
- policy_number = Vertragsnummer / Versicherungsscheinnummer
- document_date = Dokumentdatum

Format:
DD.MM.YYYY

- amount = jährlicher Beitrag / Sparbeitrag

Gemeint ist der eingezahlte Beitrag pro Jahr.

Nicht die prognostizierte Rente.
Nicht der Rückkaufswert.
Nicht das garantierte Kapital.

amount muss eine Zahl sein.
Keine Währungszeichen.

Wenn kein Beitrag gefunden wird, verwende null.

document_subtype:

contract:
Versicherungsschein, Nachtrag, Antrag, Vertragsunterlagen

annual_statement:
jährliche Mitteilung zum Stand der Rentenversicherung

cost_statement:
jährliche Kostenmitteilung

surrender_value_table:
Rückkaufswerte, beitragsfreie Werte

pension_information:
allgemeine Renteninformation

bauspar_jahresauszug:
jährlicher Kontoauszug eines Bausparvertrags

steuerbescheinigung:
Steuerbescheinigung über Kapitalerträge

Bei bauspar_jahresauszug zusätzlich (sonst null):

- interest = Guthabenszinsen
- contributions_total = Summe der Beiträge im Jahr
- opening_balance = Saldovortrag
- closing_balance = Endsaldo

Steuern (Kapitalertragssteuer, Soli) NICHT vom Kontoauszug
erfassen; diese stehen auf der Steuerbescheinigung.

Bei steuerbescheinigung zusätzlich (sonst null):

- interest = Kapitalerträge
- capital_gains_tax = Kapitalertragssteuer
- soli = Solidaritätszuschlag
- church_tax = Kirchensteuer

Beträge müssen Zahlen sein. Keine Währungszeichen.

Wenn ein Wert nicht gefunden wird, verwende "".

Dokument:

{document_text}
